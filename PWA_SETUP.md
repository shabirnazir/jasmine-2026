## 🚀 Using the Install Button

### Quick Start - Add to Topbar

Edit `components/Topbar/Topbar.js`:

```jsx
import InstallButton from "@/components/InstallButton";

// Inside your component JSX, add:
<InstallButton />;
```

### Using the PWA Hook

In any client component ('use client'):

```jsx
"use client";

import { usePWA } from "@/app/PWAProvider";

export default function MyComponent() {
  const { isInstallable, isInstalled, installApp, isOnline } = usePWA();

  return (
    <div>
      {isInstallable && !isInstalled && (
        <button onClick={installApp}>Install App</button>
      )}
      {isInstalled && <p>✅ App is installed!</p>}
      {!isOnline && <p>⚠️ You're offline</p>}
    </div>
  );
}
```

## 🧪 Testing the PWA

### 1. Development Testing

```bash
npm run dev
# Open http://localhost:3000
```

### 2. Verify in DevTools

1. Open DevTools (F12)
2. Go to **Application** tab
3. Check **Manifest** - should show app details
4. Check **Service Workers** - should be registered
5. Check **Cache Storage** - should have "jasmine-store-v1"

### 3. Test Install Button

The install button appears when:

- ✅ App is NOT installed
- ✅ Valid manifest is loaded
- ✅ Service worker is registered
- ✅ Browser supports `beforeinstallprompt`

### 4. Build for Production

```bash
npm run build
npm start
# Open http://localhost:3000
```

### 5. Mobile & Desktop Installation

- **Android Chrome**: Click "Install app" in address bar menu
- **iOS Safari**: Tap Share → Add to Home Screen
- **Windows/Mac Chrome**: Click install icon in address bar

### 6. Test Offline Mode

1. DevTools → Network tab
2. Check "Offline" checkbox
3. Navigate to previously visited pages
4. They load from cache
5. API calls handled gracefully

## 🎨 Customize Icons

### Option 1: Keep Placeholder SVG Icons

Current setup uses simple SVG placeholders - ready to use!

### Option 2: Replace with Your Logo

1. **Generate icons** using:
   - [PWA Builder](https://www.pwabuilder.com/) - easiest
   - [Favicon Generator](https://www.favicon-generator.org/)
   - Design in Figma and export

2. **Save as SVG or PNG** to:
   - `public/icons/icon-192x192.svg` (or .png)
   - `public/icons/icon-512x512.svg` (or .png)
   - `public/icons/maskable-192x192.svg` (or .png)
   - `public/icons/maskable-512x512.svg` (or .png)

3. **If using PNG**: Update `public/manifest.json`:
   - Change `"type": "image/svg+xml"` to `"type": "image/png"`
   - Update file extensions from `.svg` to `.png`

## 🔍 Troubleshooting

### Install button not showing?

```
✓ Run DevTools → Application → Manifest
✓ Check manifest is valid JSON
✓ Look for Console errors
✓ Ensure icons are accessible
✓ Try `npm run build && npm start` (production mode)
```

### Service Worker not registering?

```
✓ Check Console tab for errors
✓ Go to DevTools → Application → Service Workers
✓ If empty, clear site data: DevTools → Application → Clear site data
✓ Refresh page
```

### Offline page not showing?

```
✓ DevTools → Network → check "Offline"
✓ Visit a new page
✓ Should show offline.html (beautiful page)
```

### Install prompt appears once then never again?

```
✓ Normal behavior - browser shows once per visit
✓ User can still access via browser menu
✓ InstallButton component manages this
```
